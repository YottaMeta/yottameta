## 产品是什么

**YottaMeta（秭元）** 是一款面向开发者与知识工作者的 **本地 AI 智能体桌面应用**——不只写代码，办公文档、调研整理、知识库、自动化都能在同一工作区闭环：

- 绑定你的 **项目或工作文件夹**（本地或远程 SSH），在长期任务里持续读写、跑终端、交付文档与产物  
- **透明执行**：浏览、读文件、改代码、跑命令都会在时间线里可见，不是黑盒「说做完了」  
- **技能与工具**：内置大量可启用技能包，支持 MCP 扩展、网页浏览、知识库、自动化等  
- **数据在本地**：对话、工作空间、配置落在本机；模型 API 由你自行配置（OpenAI 兼容 / 本地模型等）

适合：开发闭环、调研写作、知识整理、办公文档、长会话项目推进——**不是**另一个在线聊天网页。

## 核心功能

### 工作空间与开发闭环

- **本地工作空间**：绑定项目目录，Agent 在该上下文中读改文件、理解结构  
- **远程工作空间（SSH）**：连接远程 Linux 主机，在远端目录执行开发任务  
- **游离会话**：不绑项目的独立任务空间，适合临时调研与写作  
- **交互终端**：多标签 PowerShell，命令输出与 Agent 操作可对照查看  
- **产物与审批**：写文件、删改、危险命令等可按策略审批或透明执行  

### 技能（Skills）

- 设置 → **技能**：搜索、备货、启用数百条内置技能（开发、办公、浏览、文档等）  
- 对话中输入 **`/技能名`** 或从列表选用，按场景注入工作流  
- 支持从技能市场在线安装扩展包（按需）  

### 模型与对话

- 设置 → **模型**：配置 OpenAI 兼容 API、本地 Ollama 等  
- **透明 ReAct**：浏览 → 读 → 写 → 跑，全过程可在消息区展开查看  
- **上下文压缩**：长项目自动压缩历史，降低 token 消耗  
- **思维辅助**（可选）：开工契约、方案分叉、人话时间线、收工锚点等结对协作能力  

### 扩展能力

| 模块 | 说明 |
|------|------|
| **MCP** | 连接 Model Context Protocol 服务，扩展 Agent 工具 |
| **联网搜索** | Tavily / DuckDuckGo 等（自行配置密钥） |
| **网页浏览** | 内置浏览组件，支持截图、页面阅读（gstack 技能族） |
| **知识库** | 本地知识空间、批量学习、检索增强 |
| **自动化** | 定时/触发运行 Agent 任务 |
| **安全扫描** | 对工作空间做基础安全规则扫描 |
| **远程 SSH** | 双工作空间隔离、离线门禁 |

> 零售版 **不含**「智能体工坊」（批量导出自定义安装包）；该能力仅面向作者/OEM 许可。

## 快速上手（5 分钟）

1. **安装并首启** — 等待技能种子加载完成（侧栏「技能」不再显示安装中）  
2. **配置模型** — 设置 → 模型 → 填入 API Key 与 Base URL → 测试连接  
3. **绑定工作空间** — 左侧「工作空间」→ 选择本地项目文件夹  
4. **新建会话** — 描述你的任务，例如：「阅读 README，列出待改项并修一个编译错误」  
5. **查看过程** — 展开时间线步骤，确认读写了哪些文件、跑了哪些命令  

**常用入口**

| 你想… | 去哪里 |
|--------|--------|
| 改模型 / 超时 | 设置 → 模型 / Agent 运行 |
| 找技能 | 设置 → 技能，或输入 `/` |
| 接 MCP | 设置 → MCP |
| 看许可 | 设置 → 许可与激活 |
| 查更新 | 设置 → 关于（REL-2 上线后） |

## 安全与隐私

- 安装包 **不含开发者工具（F12）**，防止终端用户轻易篡改前端或窥探内部接口  
- 许可状态、试用数据 **签名落盘**；核心文件篡改会触发完整性门控  
- 模型请求发往 **你配置的 API 服务商**；YottaMeta 不代持你的模型密钥  
- 工作区文件、数据库、配置存储在 **本机用户目录**（`%APPDATA%` 下应用数据路径）

## 常见问题

**Q：SmartScreen 拦安装？**  
A：未购买代码签名证书前属正常现象；点「更多信息」→「仍要运行」。正式版将逐步接入 Authenticode 签名。

**Q：安装后一闪就没了？**  
A：多为 **WebView2 未装好**；请联网重装，或手动安装 [Microsoft WebView2 运行时](https://developer.microsoft.com/microsoft-edge/webview2/)。

**Q：Release 里为什么有 Source code zip？**  
A：GitHub 自动生成，内容仅为 **本 README**，不是 YottaMeta 工程源码。

**Q：和 IDE 里的 AI 助手有什么区别？**  
A：YottaMeta 是 **绑定工作区的桌面智能体运行时**——长任务、透明时间线、终端与记忆、技能/MCP 在同一应用里闭环；IDE 内助手擅长行内编辑与代码索引。可按场景配合使用，也可以单独用 YottaMeta 推进项目级任务。
